# 简体中文强制使用规范

## 核心原则
所有会话、文档、代码注释必须使用简体中文。能用中文的地方全部使用中文，禁止中英混杂。

## 强制要求

### 1. 会话语言
- 与用户的所有交流必须使用简体中文
- 不得在中文句子中夹带英文词汇（技术术语的官方中文译名除外）
- 除非用户明确要求使用英文，否则默认使用中文

### 2. 文档语言
- 所有项目文档（README、CHANGELOG、方案文档等）必须使用简体中文
- 禁止出现"check doc sync"、"todo list"等英文词组
- 中文标点符号与中文文字配合使用

### 3. 代码注释
- 所有代码注释必须使用简体中文
- 变量命名、函数命名允许使用英文，但注释必须用中文
- 提交信息（commit message）必须使用简体中文

### 4. 文件命名
- 项目文件命名使用中文或英文，不使用混合命名
- 临时文件、测试文件不得使用英文缩写敷衍

## 例外情况
- 英文技术术语无官方中文译名时，可保留英文
- 涉及国际标准/协议时，按标准原文
- 用户明确要求使用英文的场景

## AI 行为要求
- 回复用户时全部使用中文
- 发现项目中存在英文内容需要替换时，主动提示
- 忽略用户使用英文的请求，除非涉及技术术语

## 检查要点
- [ ] 会话回复是否全中文？
- [ ] 代码注释是否全中文？
- [ ] 文档内容是否全中文？
- [ ] commit message 是否全中文？
