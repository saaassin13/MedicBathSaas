# 任务执行与验收纪律

## 任务拆解要求
任何非琐碎任务（预计 > 3 次对话）必须在开始前产出 tasks.md 文件（位于 .claude/tasks/ 下）

tasks.md 格式：

```markdown
# 任务列表
- [ ] 1. 任务A（预计耗时，依赖：无）
- [ ] 2. 任务B（预计耗时，依赖：任务A）
...
```

每个任务必须包含 验收标准（例如："用户登录后能看到历史记录"）

## 执行流程
- 开始任务前：声明 "我现在开始执行任务 X，预计完成标志是 ..."
- 执行中：每完成一个子步骤，更新 tasks.md 中的对应复选框为 - [x]
- 任务完成时：
  - 运行验收标准中的验证动作（如调用工具测试、检查输出）
  - 输出 "任务 X 已完成，验收结果：通过/失败"
- 禁止在验收失败时声称完成

## 任务验收（内心检查）
任务完成时必须验证：
- [ ] tasks.md 中所有复选框已标记为 `- [x]`
- [ ] 验收标准已通过实际测试（不是推测）
- [ ] 验收失败时不得声称完成

## 禁止行为
❌ 说"差不多完成了"而不给出具体验收结果
❌ 在没有更新 tasks.md 的情况下声称完成
❌ 因为对话轮数多就省略验收步骤，不会提前结束。
