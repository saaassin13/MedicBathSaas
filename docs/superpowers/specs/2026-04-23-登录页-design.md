## 4. 功能需求

### 4.1 表单字段

| 字段 | 类型 | 验证规则 | 错误提示 |
|------|------|---------|---------|
| 用户名 | text | 必填，最少3个字符 | 请输入用户名 |
| 密码 | password | 必填，最少6个字符 | 请输入密码 |

### 4.2 交互状态

#### 输入框状态
| 状态 | 样式变化 |
|------|---------|
| 默认 | 灰色边框 #E0E0E0 |
| 聚焦 | 橙色边框 #FF7300，图标同步变色 |
| 错误 | 红色边框 #FF4D4F，错误提示文字 |

#### 登录按钮状态
| 状态 | 样式变化 |
|------|---------|
| 默认 | 橙色背景 #FF7300 |
| 悬停 | 背景加深 #E66600，阴影上移 |
| 点击中 | 显示"登录中..."，禁用重复点击 |
| 禁用 | 灰色背景，不可点击 |

### 4.3 登录流程

```
用户输入用户名/密码
    ↓
点击登录按钮 或 按下回车键
    ↓
前端验证（空值、格式检查）
    ↓
验证失败 → 显示错误提示
    ↓
验证通过 → 调用登录接口
    ↓
登录成功 → 跳转到用户管理首页
    ↓
登录失败 → 显示错误提示，输入框边框变红
```

## 5. 技术选型

### 5.1 前端框架
- **React 18** + TypeScript
- 理由：生态成熟，组件复用性强，适合多页面企业管理系统

### 5.2 样式方案
- **TailwindCSS** + CSS Module 混合
- 理由：TailwindCSS 快速实现毛玻璃效果和响应式布局

### 5.3 UI 组件库
- **Ant Design 5**
- 理由：企业级 B 端设计语言与设计图高度契合

### 5.4 路由
- **React Router v6**
- 理由：React 生态标准路由方案
