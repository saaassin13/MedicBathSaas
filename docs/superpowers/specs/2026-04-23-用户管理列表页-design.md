## 5. 功能需求

### 5.1 用户列表

| 功能 | 描述 |
|------|------|
| 数据展示 | 显示用户名、真实姓名、邮箱、联系电话、部门、状态、权限、创建时间 |
| 搜索 | 支持按用户名、姓名、手机号搜索 |
| 分页 | 默认每页20条，支持切换每页条数 |

### 5.2 表格列定义

| 列名 | 宽度 | 省略规则 |
|------|------|---------|
| 勾选框 | 48px | 固定 |
| 用户名 | 120px | 允许省略 |
| 真实姓名 | 100px | 允许省略 |
| 邮箱 | 180px | 允许省略 |
| 联系电话 | 130px | 固定 |
| 部门 | 120px | 允许省略 |
| 状态 | 100px | 固定，Switch开关 |
| 权限 | 自适应 | 省略+Tooltip（超长显示...，悬停显示完整） |
| 创建时间 | 160px | 固定，格式 YYYY-MM-DD HH:mm |
| 操作 | 120px | 固定，[编辑][删除]链接 |

### 5.3 新增用户

| 字段 | 类型 | 验证规则 |
|------|------|---------|
| 用户名 | text | 必填，3-20字符，唯一性校验 |
| 真实姓名 | text | 必填，2-20字符 |
| 电话 | text | 必填，11位手机号 |
| 邮箱 | text | 必填，邮箱格式 |
| 密码 | password | 必填，6位以上 |
| 状态 | switch | 默认启用 |
| 权限 | tree | 必填，选择权限节点 |

### 5.4 编辑用户

- 复用新增用户弹窗
- 表单预填充用户数据
- 密码字段留空（不显示）

### 5.5 删除用户

| 场景 | 行为 |
|------|------|
| 单行删除 | 点击"删除"链接，弹出确认框 |
| 批量删除 | 选中多项后，顶部显示浮动操作栏，点击"删除"弹出确认框 |

### 5.6 状态切换

- 点击 Switch 开关
- 调用 API 更新状态
- 显示 loading 状态
- 更新成功后刷新列表

## 6. 交互逻辑

### 6.1 批量选择交互

```
用户勾选表格行（至少1项）
    ↓
表格上方出现浮动操作栏
    ↓
显示"已选中 X 项 [取消] [删除]"
    ↓
点击"取消" → 清除所有勾选，隐藏操作栏
点击"删除" → 弹出确认框
    ↓
确认删除 → 调用批量删除API → 成功后刷新列表
取消删除 → 关闭确认框，保留勾选状态
```

### 6.2 搜索交互

```
用户输入搜索关键词
    ↓
防抖 500ms
    ↓
触发搜索请求
    ↓
更新表格数据
```

### 6.3 侧边栏交互

```
点击菜单项 → 切换到对应页面
点击带下拉箭头的菜单 → 展开二级菜单
点击左下角折叠图标 → 收起/展开侧边栏
```

## 7. 状态定义

### 7.1 用户状态

```typescript
interface User {
  id: string;
  username: string;
  realName: string;
  email: string;
  phone: string;
  department: string;
  status: 'enabled' | 'disabled';
  permissions: PermissionNode[];
  createdAt: string;
}

interface PermissionNode {
  key: string;
  title: string;
  children?: PermissionNode[];
}
```

### 7.2 页面状态

```typescript
interface UserManageState {
  users: User[];
  loading: boolean;
  selectedIds: string[];
  searchKeyword: string;
  pagination: {
    page: number;
    pageSize: number;
    total: number;
  };
  modalVisible: boolean;
  modalMode: 'create' | 'edit';
  editingUser: User | null;
}
```
