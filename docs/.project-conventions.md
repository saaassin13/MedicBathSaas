# 项目路径约定

> 本文件是项目路径的**唯一真相源**。rules 中引用路径时使用此处定义的目的标识，不硬编码路径。
> 换开发方法/规划时，只需修改此文件，rules 无需变动。

---

## 文档位置

| 用途 | 路径 | 说明 |
|------|------|------|
| superpowers 规格 | `docs/superpowers/specs/` | 需求、功能说明 |
| superpowers 计划 | `docs/superpowers/plans/` | 实现计划、任务分解 |
| 技术决策 | `docs/decisions/` | ADR 记录 |
| Bug 修复 | `docs/bugfixes/` | Bug 记录 |
| 教训记录 | `docs/lessons-learned/` | 陷阱记录 |
| API 文档 | `docs/api/` | 接口文档 |
| 数据库 | `docs/db/` | Schema、迁移 |
| 配置说明 | `docs/configuration.md` | 配置项说明 |
| 架构设计 | `docs/design/` | 架构图、流程图 |
| Changelog | `CHANGELOG.md` | 变更记录 |

---

## 路径用途速查

- **specs** — 功能要做什么（规格）
- **plans** — 怎么实现（计划/任务）
- **decisions** — 为什么这样选（决策）
- **bugfixes** — 怎么修的（bug）
- **lessons-learned** — 踩过什么坑（教训）

---

## 如何在 rules 中引用

使用目的标识而非硬编码路径：

```
[superpowers-plans]: 见 docs/.project-conventions.md → superpowers 计划
[decisions]: 见 docs/.project-conventions.md → 技术决策
[bugfixes]: 见 docs/.project-conventions.md → Bug 修复
```
